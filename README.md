A Node.js Express server for taking screenshots of the web.

This was originally an AWS Lambda function.
We no longer test or maintain serverless functionality, but it probably would still work, with minimal changes.
If you want to run this serverless, see the `serverless` branch.

## Development

1. [optional] Fork on GitHub
2. run `git clone https://github.com/Hubs-Foundation/photomnemonic.git` [or clone your fork]
3. run `cd photomnemonic`
4. run `npm install`
5. make changes
6. run `npm test`
7. Build the Docker image & push it to your development container registry:
   1. To build locally,
      1. set the environment variable `REGISTRY_NAMESPACE` to your username on Docker Hub, or a namespace in the container registry you push to.
      2. run `npm run build`
      3. locally create and run a container with the image, using Docker Desktop or another container runtime
      4. shell into the container
      5. run `curl -I http://localhost:5000/screenshot/?url=https://example.com/` and verify the status is 200
      3. push the image to your development registry, using the tag generated by the build
   2. To build in GitHub,
      1. push your code to GitHub
      2. see [the Custom Docker Build Push](https://docs.hubsfoundation.org/github-workflows.html) action documentation
8. run `kubectl set image deployment/photomnemonic photomnemonic=$REGISTRY_NAMESPACE/photomnemonic:<tag> -n hcce` where `<tag>` is the tag generated by the build
9. run `kubectl rollout status deploy/photomnemonic -n hcce`
10. test your code:
    1. run `kubectl get pods -n hcce`
    2. run `kub exec -it photomnemonic-99999 -n hcce -- sh` where `photomnemonic-99999` is the name of the pod
    3. In the pod, run `curl -I http://localhost:5000/screenshot/?url=https://example.com/` and verify the status is 200
